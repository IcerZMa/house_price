# 永豐 AI GO 競賽 - 攻房戰：房價預測模型優化與資料處理策略

本專案參加由永豐銀行與 T-Brain 舉辦之 [「永豐 AI GO 競賽 - 攻房戰」](https://tbrain.trendmicro.com.tw/Competitions/Details/30)。
在 972 支參賽隊伍中，最終取得 **Private Score 8.011811（排名第 40 名，Top 5%）**。

## 核心資料處理策略

在房地產預測中，「地點 (Location)」決定了價格的上限與下限。本專案除了使用初始提供的基本欄位外，核心邏輯在於**量化初始資料中不存在的地段價值**。

### 1. 自主構思：基於空間半徑的生活機能增強
我們觀察到初始資料僅提供行政區與座標，缺乏對「生活機能」的直接描述。基於「房價與鄰里便利性高度正相關」的假設，我們自主開發了以下特徵：
* **理由與動機**：房價不僅取決於房子本身，更取決於其周邊設施。我們認為半徑範圍內的機能密度能直接反應該地區的「開發程度」與「地段價值」。
* **具體作法**：透過經緯度運算，自主抓取房屋位置特定半徑範圍內的小學、國中、高中及醫療院所等設施數量。
* **發現與成果**：這是初始資料中沒有的特徵，也是本專案最重要的發現。實驗證明，高密度的便利設施與醫療資源對單價有極強的預測力，成功彌補了原始數據在環境描述上的真空。


### 2. 統計優化：解決房價高度偏態問題
在 EDA 過程中，我們發現原始房價數據具有**高度偏態 (High Skewness)** 的特性，即少數高價物件與多數平價物件的分佈極不均勻。
* **理由**：房價分佈通常呈現「長尾現象」，若直接進行回歸訓練，模型容易被高價極端值誤導，導致對大多數普通房產的預測精確度下降。
* **作法解釋**：對目標變數 `單價` 進行 **Log Transformation (對數轉換)**。
* **效果**：轉換後數據趨於常態分佈（Normal Distribution），降低了偏度對模型的影響，顯著提升了訓練的穩定性與收斂速度。

### 3. 進階特徵轉換工程
* **目標編碼 (Target Encoding)**：針對 `縣市`、`鄉鎮市區` 等地理欄位，計算其對應單價的**中位數 (Median)** 並進行映射。相較於一般編碼，映射中位數能將類別轉化為具備「價格含義」的數值等級。
* **類別統計映射**：針對 `主要用途` 與 `主要建材` 亦採用單價中位數映射，建立類別與價格間的強連結。

## 模型優化策略

本專案不依賴單一模型，而是透過堆疊（Stacking）與加權融合（Blending）來提升泛化能力。

### 1. 多層次堆疊架構 (Stacking Strategy)
建構兩組互補的 `StackingCVRegressor` 結構：
* **多樣化基底學習器**：整合包含 KNN、Ridge、SVR 以及強大的樹模型如 XGBoost、LightGBM 與 Random Forest。
* **元學習器 (Meta-Regressor)**：分別選用 **CatBoost** 與 **XGBoost** 作為第二層的元學習器。CatBoost 在處理映射後的類別特徵效果極佳，能有效進一步收斂誤差。

### 2. 權重搜尋與最終融合 (Weighted Blending)
* **自動化權重優化**：利用驗證集進行**暴力搜尋 (Brute Force)**，找出各預測模型結果的最佳權重組合，而非單純平均。
* **二階段融合**：將最終加權預測結果與其他穩定輸出進行二階段融合，確保模型在 Private Leaderboard 上的穩健表現。
